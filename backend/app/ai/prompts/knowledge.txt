【任务】
请阅读以下材料，为「{{category_name}}」品类的客服培训提炼一份结构化知识库。
材料按案例类型分类：优秀成交案例、失败丢单案例、普通案例。

【优秀成交案例】（客服做得好，成功促成交易）
{{excellent_materials}}

{{normal_materials}}

输出严格的 JSON（不要任何解释文字、不要 markdown 代码块标记），结构如下：

{

  "product_summary": "该品类产品的一句话介绍",
  "required_questions": [

  "common_objections": [

  ],
  "recommended_responses": [
    {
      "scenario": "场景描述，如：客户问起订量",
      "guideline": "标准应答要点，如：先确认产品规格，再说明起订量与价格阶梯"
    }
  ],
  "product_specs": {
    "材质": "可选材质清单",

【注意事项】
1. required_questions 是评分的核心依据，请尽量完整列出客服必须确认的事项。
2. success_patterns 只从优秀案例中提炼，总结可复制的成功话术和技巧。
3. failure_patterns 只从失败案例中提炼，总结高频失误和丢单原因，用于培训预警。
4. 如果某类案例为空（显示"无优秀案例"或"无失败案例"），对应数组留空 []。
5. 如果材料中没有某项信息，对应字段留空字符串或空数组，不要编造。
6. 保持客观，只提炼材料中真实出现的信息。
7. 直接输出 JSON，第一个字符必须是 {。
